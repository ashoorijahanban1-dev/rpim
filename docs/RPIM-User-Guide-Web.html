USER GUIDE · نسخهٔ ۱

RPIM ◆ دفترچه راهنمای کاربری

راهنمای کامل صاحب برند و اپراتور — از ساخت حساب تا انتشار زمان‌بندی‌شده، گزارش کلیک‌ها و خروجی کامل داده. هیچ‌چیز بدون تاییدِ شما منتشر نمی‌شود.

داشبورد: app.rpim.ir API: api.rpim.ir وضعیت محصول: MVP کامل (M0–M10)
در یک نگاه شروع سریع حساب آنبوردینگ مغز برند بریف صف تایید انتشار گزارش‌ها خروجی داده ایمنی اپراتور عیب‌یابی واژه‌نامه
۰۱ — OVERVIEW

سیستم در یک نگاه

RPIM یک بازاریاب هوشمند عامل‌محور برای برندهای فارسی‌زبان است که یک حلقهٔ بسته را می‌گرداند:

مغز برند (دانش شما) ← تولید محتوا ← کنترل کیفیت ← تایید انسانی شما ← انتشار زمان‌بندی‌شده در پیام‌رسان‌ها ← اندازه‌گیری کلیک و هزینه ← یادگیری از رفتار شما
  • هیچ‌چیز بدون تایید انسان منتشر نمی‌شود. هر محتوا از صف تایید می‌گذرد.
  • هر ادعا باید منبع داشته باشد. متن‌های بی‌منبع پرچم می‌خورند.
  • داده مال شماست. با یک کلیک همه‌چیز را می‌برید.
۰۲ — QUICK START

شروع سریع — ده دقیقهٔ اول

  1. ثبت‌نام در /register — نام برند + ایمیل + رمز (حداقل ۸ نویسه)
  2. مصاحبهٔ آنبوردینگ را کامل کنید و «تایید نهایی و ساخت پروفایل برند» را بزنید
  3. به مغز برند خوراک بدهید: متن معرفی، صفحهٔ سایت، کاتالوگ PDF
  4. اولین بریف را بسازید — سیستم پیش‌نویس تولید می‌کند
  5. در صف تایید: «بررسی کیفیت» ← ویرایش در صورت نیاز ← «تایید»
  6. جاب انتشار بسازید (کانال + چت + کد کمپین) — حداکثر ۳۰ ثانیه بعد ارسال می‌شود
  7. آخر ماه گزارش ماهانه را ببینید: پست‌ها، کلیک‌ها، هزینهٔ مدل
۰۳ — ACCOUNT

حساب و ورود

با ثبت‌نام، «برند» (tenant) شما ساخته می‌شود؛ دادهٔ هر برند از برندهای دیگر کاملاً ایزوله است. اگر توکن منقضی شود خودکار به صفحهٔ ورود برمی‌گردید. خطای «email already registered» یعنی این ایمیل حساب دارد — وارد شوید.

۰۴ — ONBOARDING

مصاحبهٔ آنبوردینگ و پروفایل برند

کیفیت همهٔ محتواهای آینده به کیفیت همین پاسخ‌ها بسته است.

فیلدچه بنویسید
لحن برندچند جمله: رسمی/صمیمی؟ شوخ/جدی؟ با مثال
پرسوناهاهر مخاطب هدف در یک خط («مدیر فروش ۳۰-۴۵ ساله»)
واژه‌نامههر خط یک «اصطلاح: توضیح» — واژه‌های اختصاصی برند
ادعاهای مجازهر خط یک ادعای مستند
ادعاهای ممنوعچیزهایی که هرگز نباید گفته شود
خطوط قرمزموضوعات ممنوعه (سیاست، رقیب خاص، …)

«ذخیره پیش‌نویس» یعنی بعداً ادامه می‌دهید؛ «تایید نهایی» پروفایل برند را می‌سازد. بعد از تکمیل هم می‌شود ویرایش و دوباره نهایی کرد.

۰۵ — BRAND BRAIN

مغز برند

هرچه اینجا بریزید، خوراک تولید محتوا و راستی‌آزمایی ادعاهاست. سه راه ورود:

  • متن مستقیم — عنوان + متن (معرفی، لیست قیمت، سؤالات متداول…)
  • کرال سایت — آدرس صفحه + حداکثر تعداد صفحه (۱ تا ۱۰)
  • PDF — آپلود کاتالوگ و بروشور
قیمت‌ها و اعداد مهم را حتماً وارد مغز کنید — عددی که در مغز نباشد، در محتوا «⚠️ عدد بدون منبع» می‌خورد.

آپلود تکراری همان سند دوباره پردازش نمی‌شود؛ «جستجوی مغز برند» همان بازیابی‌ای است که موقع تولید محتوا هم رخ می‌دهد (۵ قطعهٔ مرتبط).

۰۶ — BRIEF

تولید محتوا — بریف

فیلدمثال
هدف«معرفی طعم جدید»
مخاطب«جوانان ۱۸-۲۵»
کانالtelegram / bale / eitaa

با ثبت بریف: ۵ قطعهٔ مرتبط از مغز برداشته می‌شود ← با لحن و خطوط قرمز پروفایل، پیش‌نویس ساخته می‌شود ← به صف تایید می‌رود.

۰۷ — APPROVAL QUEUE

صف تایید — قلب سیستم

  • «بررسی کیفیت» — تطبیق ادعاها با مغز برند، حساسیت‌ها، مشخصات کانال. پرچم هشدار فقط اطلاع می‌دهد؛ پرچم مسدود یعنی «⛔ بازبینی انسانی الزامی» — اول با ویرایش رفعش کنید.
  • «⚠️ عدد بدون منبع» — عدد را اصلاح کنید یا سندش را به مغز اضافه کنید.
  • ویرایش در همان کارت + «ذخیره نسخه ویرایش‌شده»
  • تایید ← آمادهٔ انتشار · رد با دلیل (لحن / واقعیت / حساسیت / سلیقه) + یادداشت
🧠 چرا دلیلِ رد مهم است؟ هر تایید، ویرایش و ردِ شما به‌عنوان «سیگنال یادگیری» (لاگ شاگرد) ثبت می‌شود — پایهٔ شخصی‌سازی مدل روی سلیقهٔ برند شما. باارزش‌ترین دادهٔ سیستم همین است.

چرخهٔ وضعیت: پیش‌نویس ← (ویرایش) ویرایش‌شده ← تاییدشده ← ارسال‌شده · یا: ردشده

۰۸ — PUBLISH

انتشار

فقط پیش‌نویس‌های تاییدشده یا ویرایش‌شده قابل انتشارند. ساخت جاب (فعلاً از API؛ فرم داشبورد در راه است):

POST https://api.rpim.ir/publish/jobs
{
  "draft_id": "...",
  "channel": "telegram",                     // یا bale / eitaa
  "chat_id": "@yourchannel",
  "campaign_code": "yalda1404",              // اجباری
  "scheduled_at": "2026-07-10T18:00:00Z",    // اختیاری
  "landing_url": "https://shop.example.com", // اختیاری — UTM خودکار
  "image": {"template": "announce", "size": "square"}  // اختیاری
}
  • کد کمپین اجباری است — کلید اتصال پست به کلیک در گزارش‌ها.
  • متن در لحظهٔ ساخت جاب منجمد می‌شود — دقیقاً همان چیزی که تایید کردید ارسال می‌شود.
  • UTM خودکار: utm_source=کانال · utm_medium=social · utm_campaign=کد کمپین
  • پست تصویری: قالب‌های announce / quote / product در سایزهای square / story / wide با فونت فارسی
  • موتور ارسال هر ۳۰ ثانیه جاب‌های سررسید را می‌فرستد؛ خطای موقت یعنی ماندن در صف و تلاش دوباره — نه گم می‌شود، نه دوبار می‌رود.
⚙️ تا وقتی اپراتور PUBLISH_MODE=live را ست نکرده، ارسال‌ها «خشک» اجرا می‌شوند و جایی واقعاً پست نمی‌رود. برای انتشار واقعی، توکن بات هر کانال هم باید در env سرور باشد.
۰۹ — REPORTS

گزارش ماهانه

ماه را به شکل YYYY-MM وارد کنید:

  • پیش‌نویس‌ها: ساخته / تایید / ویرایش / رد
  • انتشار: در صف، ارسال‌شده، تفکیک هر کانال
  • کمپین‌ها: به‌ازای هر کد: جاب، ارسال موفق، کلیک
  • هزینهٔ مدل‌ها: مجموع دلاری + تفکیک هر ارائه‌دهنده

مسیر «پست ← کلیک ← صفحهٔ فرود» با همان کد کمپین سرتاسر قابل ردیابی است.

۱۰ — EXPORT

خروجی کامل داده‌ها

صفحهٔ «خروجی کامل داده‌ها» ← «دانلود همه داده‌ها» ← یک فایل JSON شامل: پروفایل برند، پاسخ‌های آنبوردینگ، اسناد و قطعات مغز، پیش‌نویس‌ها با نتایج کیفیت، لاگ کامل یادگیری، و جاب‌های انتشار.

دادهٔ شما، بدون قید و شرط، مال شماست.
۱۱ — SAFETY

ایمنی و کنترل انتشار

سکوت برند (دست شما)

POST /governance/silence   {"active": true, "reason": "..."}

جاب‌های در صف هم می‌ایستند — چک داخل خودِ موتور ارسال است. ازسرگیری فقط دستی است (همان فراخوان با active:false).

کنترل‌های سراسری (فقط اپراتور)

  • رویداد ملی: سیگنال فید، سکوتِ همهٔ برندها را خودکار فعال می‌کند؛ فید نمی‌تواند بردارد — فقط اپراتور.
  • کلید قطع: توقف همهٔ صف‌های انتشار در کمتر از ۵ ثانیه.
